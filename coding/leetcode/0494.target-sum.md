# Key insights
* Recursive divide-and-conquer exploring all +/- combinations for each number
* Exponential time O(2^n) due to branching factor of 2 at each decision point
* Space complexity O(n) from recursion call stack depth
* Can be optimized with memoization to avoid redundant subproblem calculations

# Code
```javascript
var findTargetSumWays = function(nums, target) {
    if (nums.length === 0) {
        return target == 0 ? 1 : 0;
    }
    return findTargetSumWays(nums.slice(1), target + nums[0]) + findTargetSumWays(nums.slice(1), target - nums[0]);
};

```
```javascript
var findTargetSumWays = function(nums, S) {
    const sum = nums.reduce((memo, n) => memo + n, 0);
    if (sum < S || (sum + S) % 2 == 1) {
        return 0;
    }
    const target = (sum + S) / 2;

    const dp = [...Array(target + 1)].fill(false);
    dp[0] = 1;
    
    for (const num of nums) {
        for (let i = target; i >= num; i--) {
            dp[i] = dp[i] + dp[i - num];
        }
    }
    
    return dp[target];
};
```

# Complexity
**Solution 1:**
**Time:** O(2^n) - each element has 2 choices (+ or -) creating a binary tree of recursive calls
**Space:** O(n) - recursion depth equals array length, plus O(n) for each slice operation

**Solution 2:**
**Time:** O(n * target) - nested loops iterate through n numbers and target sum values
**Space:** O(target) - DP array of size target+1 to store intermediate results
