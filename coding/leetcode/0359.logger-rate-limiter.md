# Logger Rate Limiter
[359. Logger Rate Limiter](https://leetcode.com/problems/logger-rate-limiter/)

Design a logger system that receives a stream of messages along with their timestamps. Each unique message should only be printed at most every 10 seconds.

Implement the Logger class with `shouldPrintMessage(int timestamp, string message)` that returns true if the message should be printed at the given timestamp.

# Key insights
* Use a map to store the last timestamp each message was printed
* If the message hasn't been seen or the time difference is >= 10, allow it and update the timestamp
* Otherwise, reject it

# Code
```javascript
Logger.prototype.shouldPrintMessage = function(timestamp, message) {
    if (!this.map.has(message)) {
        this.map.set(message, timestamp);
        return true;
    }
    
    const lastTimestamp = this.map.get(message);
    if (lastTimestamp + 10 > timestamp) {
        return false;
    }

    this.map.set(message, timestamp);
    return true;
};
```

# Complexity
**Time:** O(1) - Each call is a constant-time map lookup and insertion.
**Space:** O(n) - Where n is the number of unique messages stored.
