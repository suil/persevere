# Kth Smallest Element in a BST
[230. Kth Smallest Element in a BST](https://leetcode.com/problems/kth-smallest-element-in-a-bst/)

Given the root of a binary search tree, and an integer k, return the kth (1-indexed) smallest element in the tree.

Example 1:

Input: root = [3,1,4,null,2], k = 1
Output: 1
Example 2:


Input: root = [5,3,6,2,4,null,null,1], k = 3
Output: 3

# Key insights
* In-order traversal of BST visits nodes in ascending order
* Early termination when k elements are processed optimizes performance
* Array stores visited values to track position and return kth element

# Code
```javascript
var kthSmallest = function(root, k) {
    let res = [];
    let kth = 0;
    function inorder(node) {
        if (node === null) {
            return; 
        }
        inorder(node.left);
        res.push(node.val);
        if (res.length === k) {
            kth = node.val;
            return;
        }
        inorder(node.right);
    }
    inorder(root);
    return kth;
};
```

# Complexity
**Time:** O(n) - performs in-order traversal visiting all nodes in worst case when k equals n
**Space:** O(n) - uses O(h) recursion stack space plus O(k) for result array, where h can be n in unbalanced tree
