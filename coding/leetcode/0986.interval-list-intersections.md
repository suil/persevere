# Interval List Intersections
[986. Interval List Intersections](https://leetcode.com/problems/interval-list-intersections/)

You are given two lists of closed intervals, firstList and secondList, where firstList[i] = [starti, endi] and secondList[j] = [startj, endj]. Each list of intervals is pairwise disjoint and in sorted order.

# Key insights
- Two pointers iterate through both sorted interval lists simultaneously
- Intersection is computed as [max(starts), min(ends)]; it exists only when lo <= hi
- Advances the pointer whose interval ends first, since that interval cannot intersect with any further intervals in the other list

# Code
```javascript
/**
 * @param {number[][]} firstList
 * @param {number[][]} secondList
 * @return {number[][]}
 */
var intervalIntersection = function(firstList, secondList) {
  const output = [];
  let p1 = 0;
  let p2 = 0;

  while (p1 < firstList.length && p2 < secondList.length) {
    const lo = Math.max(firstList[p1][0], secondList[p2][0]);
    const hi = Math.min(firstList[p1][1], secondList[p2][1]);
    
    if (lo <= hi) {
      output.push([lo, hi]);
    }

    if (firstList[p1][1] < secondList[p2][1]) {
      p1++;
    } else {
      p2++;
    }
  }

  return output;
};
```

# Complexity
**Time:** O(m + n) -- each interval in both lists is visited at most once
**Space:** O(m + n) -- for the output array in the worst case
