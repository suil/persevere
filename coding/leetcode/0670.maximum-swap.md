# Maximum Swap
[670. Maximum Swap](https://leetcode.com/problems/maximum-swap/)

Given a non-negative integer, you could swap two digits at most once to get the maximum valued number. Return the maximum valued number you could get.

Example 1:
Input: 2736
Output: 7236
Explanation: Swap the number 2 and the number 7.
Example 2:
Input: 9973
Output: 9973
Explanation: No swap.

# Key insights
* Map stores last occurrence of each digit for O(1) position lookup
* Greedy approach: for each position, find largest possible digit to swap with
* Iterate digits 9 down to current+1 to find optimal swap candidate first
* Early return after first beneficial swap since only one swap is allowed

# Code
```javascript
var maximumSwap = function(num) {
    const nums = num.toString().split('');
    const positionMap = new Map();

    for (let i = 0; i < nums.length; i++) {
        positionMap.set(Number(nums[i]), i);
    }

    for (let i = 0; i < nums.length; i++) { 
        for (let d = 9; d > nums[i]; d--) {
            if (!positionMap.has(d)) { continue; }
            const pos = positionMap.get(d);
            if (pos > i) {
                [nums[i], nums[pos]] = [nums[pos], nums[i]];
                return Number(nums.join(''));
            }
        }
    }
    return num;
};
```

# Complexity
**Time:** O(n) - Converting number to string array takes O(n), building the map takes O(n), and the nested loop runs at most O(n Ã— 10) = O(n) since the inner loop is bounded by 10 digits.**Space:** O(n) - Space is used for the string array and the position map, both proportional to the number of digits in the input.
