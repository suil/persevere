# Merge Sorted Array
[88. Merge Sorted Array](https://leetcode.com/problems/merge-sorted-array/description/)

Given two sorted integer arrays nums1 and nums2, merge nums2 into nums1 as one sorted array.

The number of elements initialized in nums1 and nums2 are m and n respectively. You may assume that nums1 has a size equal to m + n such that it has enough space to hold additional elements from nums2.

Example 1:
Input: nums1 = [1,2,3,0,0,0], m = 3, nums2 = [2,5,6], n = 3
Output: [1,2,2,3,5,6]

Example 2:
Input: nums1 = [1], m = 1, nums2 = [], n = 0
Output: [1]

# Key insights
* Work backwards from end to avoid overwriting elements
* Use three pointers: two for input arrays, one for final position
* Leverage existing space in nums1 for O(1) space complexity
* Compare largest remaining elements first for proper sorting

# Code
```javascript
var merge = function(nums1, m, nums2, n) {
    if (nums1.length === 0) {
        return nums1;
    }
    let index1 = m - 1, index2 = n - 1;
    let index = nums1.length - 1;
    
    while (index1 >= 0 || index2 >= 0) {
        if (index1 >= 0 && index2 >= 0) {
            if (nums1[index1] > nums2[index2]) {
                nums1[index--] = nums1[index1--];
            } else {
                nums1[index--] = nums2[index2--];
            }
        } else if (index1 >= 0) {
            nums1[index--] = nums1[index1--];
        } else if (index2 >= 0) {
            nums1[index--] = nums2[index2--];
        }
    }
    return nums1;
};
```

# Complexity
**Time:** O(m + n) - The algorithm iterates through all elements in both arrays exactly once, processing each element from right to left.**Space:** O(1) - The merge is performed in-place using only a constant amount of extra variables (index1, index2, index), without requiring additional arrays.
