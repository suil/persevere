# Minimum Domino Rotations For Equal Row
[1007. Minimum Domino Rotations For Equal Row](https://leetcode.com/problems/minimum-domino-rotations-for-equal-row/)

In a row of dominoes, tops[i] and bottoms[i] represent the top and bottom halves of the ith domino. (A domino is a tile with two numbers from 1 to 6 - one on each half of the tile.)

# Key insights
- Counts frequency of each value (1-6) in both rows and tracks how many positions have the same value on top and bottom
- A value can fill an entire row if freqA[val] + freqB[val] - countSame equals the total length
- The minimum rotations needed is the total length minus the maximum of the two frequencies for that value

# Code
```javascript
/**
 * @param {number[]} A
 * @param {number[]} B
 * @return {number}
 */
var minDominoRotations = function(A, B) {
    const freqA = [];
    const freqB = [];
    let countSame = 0;
    
    for (let i = 0; i < A.length; i++) {
        freqA[A[i]] = (freqA[A[i]] || 0) + 1;
        freqB[B[i]] = (freqB[B[i]] || 0) + 1;
        
        if (A[i] === B[i]) countSame++;
    }
    
    for (let i = 1; i < 7; i++) {
        if (freqA[i] + freqB[i] - countSame === A.length) {
            return A.length - Math.max(freqA[i], freqB[i]);
        }
    }
    
    return -1;
};

```

# Complexity
**Time:** O(n) -- single pass to count frequencies, constant-time check for values 1-6
**Space:** O(1) -- frequency arrays are bounded by 6 values
