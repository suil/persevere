# Peeking Iterator
[284. Peeking Iterator](https://leetcode.com/problems/peeking-iterator/)

Design an iterator that supports the peek operation on an existing iterator in addition to the hasNext and the next operations.

# Key insights
- Cache the next element to enable peek without consuming it
- On construction, pre-fetch the first element if available
- On next(), return the cached value and fetch the new next element
- hasNext() simply checks if there's a cached element

/**
 * // This is the Iterator's API interface.
 * // You should not implement it, or speculate about its implementation.
 * function Iterator() {
 *    @ return {number}
 *    this.next = function() { // return the next number of the iterator
 *       ...
 *    }; 
 *
 *    @return {boolean}
 *    this.hasNext = function() { // return true if it still has numbers
 *       ...
 *    };
 * };
 */

/**
 * @param {Iterator} iterator
 */
var PeekingIterator = function(iterator) {
    this.iterator = iterator;
    this._next = null
    this._hasNext = iterator.hasNext();
    if (this._hasNext) {
        this._next = iterator.next();
    }
};

/**
 * @return {number}
 */
PeekingIterator.prototype.peek = function() {
    return this._next;
};

/**
 * @return {number}
 */
PeekingIterator.prototype.next = function() {
    const result = this._next;
    this._hasNext = this.iterator.hasNext();
    if (this._hasNext) {
        this._next = this.iterator.next();
    } else {
        this._next = null;
    }
    return result;
};

/**
 * @return {boolean}
 */
PeekingIterator.prototype.hasNext = function() {
    return this._hasNext;
};

# Complexity
**Time:** O(1) for all operations (peek, next, hasNext)
**Space:** O(1) - only storing one cached element
