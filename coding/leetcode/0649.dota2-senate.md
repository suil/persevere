# Dota2 Senate
[649. Dota2 Senate](https://leetcode.com/problems/dota2-senate/)

In the world of Dota2, there are two parties: the Radiant and the Dire.

# Key insights
- Use two queues to track the positions of Radiant and Dire senators; the senator with the smaller index bans the other
- The winning senator is re-enqueued with a new index (beyond the original length) to simulate circular rounds
- The process continues until one queue is empty, determining the winning party

# Code
```javascript
/**
 * @param {string} senate
 * @return {string}
 */
var predictPartyVictory = function(senate) {
    const rQ = [];
    const dQ = [];

    for (let i = 0; i < senate.length; i++) {
        if (senate[i] === 'R') rQ.push(i);
        if (senate[i] === 'D') dQ.push(i);
    }

    let i = senate.length;
    while (rQ.length > 0 && dQ.length > 0) {
        const rPos = rQ.shift();
        const dPos = dQ.shift();

        if (rPos < dPos) {
            rQ.push(i++);
        } else {
            dQ.push(i++);
        }
    }

    if (rQ.length === 0) return "Dire";
    return "Radiant";
};
```

# Complexity
**Time:** O(n) -- each senator is processed a constant number of times across all rounds
**Space:** O(n) -- for the two queues
