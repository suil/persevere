# Find in Mountain Array
[1095. Find in Mountain Array](https://leetcode.com/problems/find-in-mountain-array/)

You may recall that an array arr is a mountain array if and only if:
- arr.length >= 3
- There exists some i with 0 < i < arr.length - 1 such that:
  - arr[0] < arr[1] < ... < arr[i - 1] < arr[i]
  - arr[i] > arr[i + 1] > ... > arr[arr.length - 1]

Given a mountain array mountainArr, return the minimum index such that mountainArr.get(index) == target. If such an index does not exist, return -1.

You cannot access the mountain array directly. You may only access the array using a MountainArray interface:
- MountainArray.get(k) returns the element of the array at index k (0-indexed).
- MountainArray.length() returns the length of the array.

Submissions making more than 100 calls to MountainArray.get will be judged Wrong Answer.

# Key insights
* Three-phase binary search: find peak, search left side, search right side
* Find the peak index first using binary search â€” if mid < mid+1, peak is to the right
* Search the ascending left half with standard binary search for the target
* Only search the descending right half if target was not found on the left (to guarantee minimum index)
* Minimize API calls by reusing the peak-finding result and short-circuiting early

# Code
```javascript
var findInMountainArray = function(target, mountainArr) {
    
    function findPeakIndex(arr) {
        let left = 0, right = arr.length() - 2;
        while (left <= right) {
            const mid = Math.floor((left + right ) / 2);
            if (arr.get(mid) < arr.get(mid + 1)) {
                left = mid + 1;
            } else {
                right = mid - 1;
            }
        }
        return left;
    }
    
    function findInLeft(arr, left, right, target) {
        while (left <= right) {
            const mid = Math.floor((left + right) / 2);
            if (target > arr.get(mid)) {
                left = mid + 1;
            } else if (target < arr.get(mid)) {
                right = mid - 1;
            } else {
                return mid;
            }
        }
        return -1;
    }

    function findInRight(arr, left, right, target) {
        while (left <= right) {
            const mid = Math.floor((left + right) / 2);
            if (target < arr.get(mid)) {
                left = mid + 1;
            } else if (target > arr.get(mid)) {
                right = mid - 1;
            } else {
                return mid;
            }
        }
        return -1;
    } 

    const peakIndex = findPeakIndex(mountainArr);
    const leftIndex = findInLeft(mountainArr, 0, peakIndex, target);
    if (leftIndex != -1) return leftIndex;

    return findInRight(mountainArr, peakIndex, mountainArr.length() - 1, target);
};
```

# Complexity
**Time:** O(log n) - Three binary searches each run in O(log n): one to find the peak, one on the ascending half, and one on the descending half.

**Space:** O(1) - Only a constant amount of extra space is used for pointers and temporary variables regardless of input size.
